  1. Ouvrir login.html
  2. Saisir l'email et le mot de passe (ou cliquer sur l'email dans la liste)

Points d'attention connexion
Scope Chef de chantier Le Chef ne voit QUE les données de son chantier. Si les données n'apparaissent pas, vérifier que user_metadata.chantier correspond exactement au nom dans la table chantiers.
Déconnexion Toujours utiliser le bouton Déconnexion (coin haut-droite) plutôt que de fermer l'onglet, pour invalider le token Supabase.

Administrateur — admin.html

Accès total au système. Validation centralisée, gestion utilisateurs, KPI globaux.

Sections du dashboard :
  • 📥Import Excel — charger des données en masse (.xlsx)
  • 💾Sauvegarde — exporter toutes les données
  • 👥Utilisateurs — créer / gérer comptes
  • 📋Rapports chantier — vue globale
  • 🔍Contrôles — inspections inopinées
  • ✅Validations — approuver/rejeter les demandes
  • 📊Gantt — avancement global
  • 📡Supabase — état de la connexion
Créer un nouveau Chef de chantier :
  1. Aller dans Utilisateurs
  2. Rôle : sélectionner chef
  3. Renseigner email + mot de passe
  4. Renseigner Chantier (correspondance exacte avec la DB)
  5. Cliquer Créer l'utilisateur
KPI Bénéfice net Affiché en vert (positif) ou rouge (négatif) sur le dashboard. Calculé dynamiquement : Recettes − Dépenses.

Comptabilité, budgets, devis/proforma, factures, rapports financiers.

Modules principaux :

  • 💹Budget global — gestion budgétaire

  • 🧾Factures — tableau + nouvelle facture
  • 📈Rapports — export PDF/Excel

RH — Ressources Humaines — rh.html

Employés, paie, congés, formations, recrutement, comptes chefs.

Modules :
  • 👤Employés — 82 actifs, filtres département/chantier
  • 🎯Recrutement — demandes + suivi
  • 🏖️Congés — demandes, dates, motifs
  • 📚Formations — plan + suivi
  • 💳Paie — génération fiches, export PDF, réajustement

Générer les fiches de paie :
  1. Section Paie → cliquer Générer fiches
  2. Les pointages sont agrégés automatiquement
  3. Tableau : employé, jours, taux, brut, avances, net

  5. Export individuel PDF ou export global
  6. Option Réajuster salaires : modal d'indexation

Chef de chantier — chef-chantier.html

Scopé à UN seul chantier. Pointage, équipe, matériaux, planning, rapports.

⚠️ Isolation par RLS Le Chef ne voit QUE les données de son chantier (filtrage automatique via user_metadata.chantier). Aucune donnée des autres chantiers n'est accessible.
Modules :
  • 🏢Mes Chantiers — statut, budget, avancement
  • 👷Mon Équipe — ouvriers affectés
  • ✍️Pointage — manuel ou QR code
  • 📅Planning — tâches, priorités
  • 🧱Matériaux — stocks + demandes
  • 📋Rapports — rapports journaliers
Pointage manuel :
  1. Section Pointage
  2. Sélectionner l'employé dans la liste déroulante
  3. Choisir la date
  4. Sélectionner le statut : Présent / Absent / Congé…
  5. Cliquer Enregistrer

Contrôleur — controleur.html

Inspections qualité/sécurité sur tous les chantiers. Accès multi-chantiers.

Modules :
  • 📋Inspections — toutes les inspections, toutes les chantiers
  • 🏆Qualité — checklist 7 critères
  • ⛑️Sécurité — checklist 7 vérifications
  • 📊Rapports — export inspections
Statut automatique Si un critère qualité ou sécurité est décoché, le statut passe automatiquement à Non conforme.
Créer une inspection :
  1. Section Inspections → Nouvelle Inspection
  2. Sélectionner le chantier dans la liste
  3. Remplir la checklist qualité (7 éléments)
  4. Remplir la checklist sécurité (7 éléments)
  5. Ajouter les observations
  6. Enregistrer — statut calculé automatiquement

Chef opérationnel
  1. [Admin] Se connecter sur admin.html → section Utilisateurs
  2. [Admin] Créer un chantier : aller dans la base Supabase ou via l'import Excel — renseigner nom, client, budget, dates
  3. [DAF] Se connecter sur daf.html → section Budget — Associer une ligne budgétaire au chantier
  4. [RH] Se connecter sur rh.html → section Comptes chefs — Créer le compte chef avec le nom exact du chantier
  5. [Admin] Section Validations — Approuver la demande de création du chef EN ATTENTE
  6. [Chef] Se connecter sur login.html avec les identifiants reçus — vérifier la redirection vers chef-chantier.html
  7. [Chef] Dashboard → vérifier que seul le bon chantier apparaît dans Mes Chantiers
Vérification clé Le Chef ne doit voir AUCUNE donnée des autres chantiers. Si c'est le cas, vérifier le champ chantier dans user_metadata.
S2

Devis Proforma → Conversion en Chantier

Durée estimée : 8 min | Rôles : DAF Admin

Créer devis
→
Soumettre
→
Validation Admin
→

Chantier créé
  1. [DAF] Section Devis & Proforma → Nouveau devis
  2. [DAF] Renseigner : client, objet des travaux, lots (désignation, quantité, PU), TVA → TTC calculé automatiquement
  3. [DAF] Enregistrer en BROUILLON → vérifier les montants
  4. [DAF] Cliquer Soumettre → statut passe à SOUMIS
  5. [Admin] Section Validations → trouver la demande de type "Devis" → Approuver
  6. [DAF] Statut passe à APPROUVÉ → envoyer au client → statut ENVOYÉ
  7. [DAF] Enregistrer la réponse client : Accepter → statut ACCEPTÉ
  8. [DAF] Cliquer Convertir en Chantier → formulaire pré-rempli (objet → nom, client, budget TTC)
  9. [Admin] Valider la conversion → le chantier est créé avec devis_id traçable
S3

Onboarding d'un nouvel employé

Durée estimée : 5 min | Rôles : RH Admin Chef

  1. [RH] Section Employés → Nouvel employé
  2. [RH] Renseigner : nom, prénom, poste, département, chantier, date embauche, salaire, type_salaire (Mensuel ou Journalier)
  3. [RH] Enregistrer — la demande est soumise à validation
  4. [Admin] Section Validations → approuver la création de l'employé
  5. [Chef du chantier concerné] Section Mon Équipe → l'employé apparaît dans sa liste
  6. [Chef] Tester le pointage : section Pointage → sélectionner le nouvel employé → enregistrer une présence
Type de salaire Bien sélectionner Mensuel ou Journalier — ce champ détermine le calcul de la paie. Ne pas laisser le système deviner.
S4

Pointage quotidien et calcul de paie

Durée estimée : 6 min | Rôles : Chef RH

  1. [Chef] Section Pointage chaque matin
  2. [Chef] Pour chaque ouvrier : sélectionner dans la liste → statut (Présent / Absent / Congé / …) → Enregistrer
  3. Répéter chaque jour ouvré du mois — les données s'accumulent dans pointage_attendance
  4. [RH] Fin de mois → section Paie → Générer fiches
  5. [RH] Tableau apparaît : employé, jours présents, taux journalier, brut, déductions CNaPS/OSTIE, net
  6. [RH] Cliquer Fiche de paie sur chaque ligne pour voir le détail complet
  7. [RH] Exporter : Export global (toutes les fiches) ou Export chantier (filtré)
S5

Demande de matériaux sur chantier

Durée estimée : 5 min | Rôles : Chef Admin

  1. [Chef] Section Matériaux — consulter l'état du stock actuel
  2. [Chef] Si stock insuffisant → Demande de matériaux
  3. [Chef] Renseigner : matériau, quantité, motif de la demande
  4. [Chef] Soumettre → demande en EN ATTENTE de validation
  5. [Admin] Section Validations → trouver la demande matériaux → Approuver ou Rejeter avec motif
  6. [Chef] Après approbation → le stock du chantier est mis à jour automatiquement
Rejet Si l'Admin rejette, il doit obligatoirement renseigner un motif. Le Chef verra le motif dans ses demandes.
S6

Inspection qualité/sécurité inopinée

Durée estimée : 7 min | Rôles : Contrôleur Admin

  1. [Contrôleur] Section Inspections → Nouvelle Inspection
  2. [Contrôleur] Sélectionner le chantier dans la liste déroulante
  3. [Contrôleur] Remplir la checklist Qualité (7 critères) :
    Cocher les éléments conformes, laisser vide les non-conformes
  4. [Contrôleur] Remplir la checklist Sécurité (7 vérifications)
  5. [Contrôleur] Ajouter des observations textuelles
  6. Le statut est calculé automatiquement : Conforme ou Non conforme
  7. [Contrôleur] Enregistrer → le rapport est soumis à l'Admin
  8. [Admin] Section Contrôles → voir l'inspection, historique, statistiques
S7

Cycle comptable mensuel

Durée estimée : 10 min | Rôles : DAF Admin

  1. [DAF] Section Comptabilité → consulter le journal (200+ lignes existantes)
  2. [DAF] Nouvelle écriture → renseigner : date, libellé, débit/crédit, catégorie, chantier
  3. [DAF] Appliquer les filtres pour vérifier les recettes / dépenses du mois
  4. [DAF] Section Budget FELANA → consulter le tableau postes × mois (prévu vs réel)
  5. [DAF] Vérifier les alertes de dépassement (seuil configurable)
  6. [DAF] Section Rapports financiers → exporter en PDF ou Excel
  7. [Admin] Dashboard → KPI Bénéfice net (Recettes − Dépenses) en vert ou rouge
S8

Calcul et export des salaires

Durée estimée : 8 min | Rôles : Chef RH

  1. Prérequis : pointages du mois enregistrés par le Chef dans pointage_attendance
  2. [RH] Section Paie → Générer fiches
  3. Calcul : brut = jours_présents × salaire_journalier (ou salaire mensuel si type Mensuel)
  4. Déductions automatiques : CNaPS + OSTIE calculés selon barème
  5. Colonne Net = brut − déductions − avances
  6. [RH] Cliquer Fiche de paie → visualiser le détail complet
  7. [RH] Export global → télécharger toutes les fiches en un fichier
  8. [RH] Pour ajuster les salaires : Réajuster salaires → modal d'indexation (en %)
  9. [RH] Export chantier → filtrer par chantier pour transmission aux chefs
QR Code Pour les salaires payés en espèces, le QR Code encode {id, nom} uniquement — jamais le montant, pour raisons de sécurité.

✅
Procédure Admin — Approuver
  1. Se connecter en Admin
  2. Section Validations → badge "X en attente"
  3. Identifier la demande dans le tableau (Type, Émetteur, Date, Objet)
  4. Cliquer Approuver (commentaire optionnel)
  5. L'effet est appliqué automatiquement selon le type
  6. La demande passe en APPROUVÉ puis est archivée
❌
Procédure Admin — Rejeter
  1. Dans le tableau Validations, trouver la demande
  2. Cliquer Rejeter
  3. Renseigner le motif de rejet (obligatoire)
  4. Confirmer
  5. L'émetteur verra le motif dans son interface
  6. La demande est archivée avec statut REJETÉ

Guide de dépannage
Connexion refusée
Vérifier email + mot de passe. Si le compte n'existe pas encore, demander à l'Admin de le créer via Admin → Utilisateurs.
Page blanche après connexion
Attendre le chargement complet. Si ça persiste : vider le cache navigateur (Ctrl+Shift+R) et se reconnecter.
Erreur 401 / RLS policy
L'utilisateur n'a pas les droits pour cette action. Vérifier que son rôle dans user_metadata est correct.
Chef ne voit aucun employé
Le champ chantier du compte chef doit correspondre EXACTEMENT au nom dans la table chantiers (respect de la casse et des espaces).
Données non chargées
Vérifier la connexion internet. Recharger la page. Si hors-ligne, les données locales PWA sont utilisées — la sync se fait au retour en ligne.
Stock/budgets non sauvegardés
Dans certaines versions, budgets et stock utilisaient localStorage. Vérifier que les migrations Supabase ont bien été exécutées (table materiels + budgets).
Pointage enregistré mais absent du calcul paie
Vérifier que les pointages sont dans pointage_attendance et non dans l'ancienne table pointage. Migration I1 requise.
suivi-chantier.html accessible sans connexion
Anomalie connue (C4) — la guard checkAuthOrRedirect() doit être ajoutée en haut du fichier. Signaler à l'admin technique.
Réinitialiser toutes les données
Exécuter dans Supabase SQL Editor :
TRUNCATE personnel, pointage_attendance, journal, salaires, rapports_chantier, controles_inopines, gantt_taches, stocks_chantier, commandes, validations, caisse RESTART IDENTITY CASCADE; puis réexécuter SEED_DATA.sql
